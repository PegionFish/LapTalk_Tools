# HWiNFO CSV 折线图导出工具

一个面向 Windows GUI 的小工具，用来读取 HWiNFO 导出的 CSV 文件，按时间戳绘制指定参数的折线图，并导出透明背景 PNG，方便接入后续开发流程。

## 功能

- 读取 HWiNFO CSV 文件
- 以从 `00:00:00` 开始累计的抽象时长作为 X 轴
- 以指定参数的数值作为 Y 轴
- 支持多选参数并在同一张图里叠加显示
- 导出透明背景 PNG
- 自动预览图表，无需手动点击刷新
- 选择 CSV 后会先列出可用参数，并在后台预载全部数值序列到内存中，后续预览与导出更流畅
- 预览与导出都会先在内存中生成 PNG，预览界面只负责显示这张原图并按窗口自适应查看
- 支持标题、线宽、网格、图例位置、纯曲线模式、时间/数值刻度文字显示、时间刻度密度滑块、前后裁剪滑块，以及不同参数的独立颜色设置
- 支持中文列名、重复列名和列名筛选，并自动识别常见 CSV 编码
- 左侧控制区支持纵向滚动，非全屏窗口下也能直接看到参数筛选与列表
- 鼠标指针停在参数列表或左侧控制区时，可直接使用滚轮滚动

## 环境

- Windows
- Python 3.10+
- `matplotlib`

安装依赖：

```powershell
pip install -r requirements.txt
```

## 启动

双击 `run_gui.bat`，或者手动执行：

```powershell
python .\main.pyw
```

## 打包 EXE

1. 确保已安装 `csv_visual/requirements.txt` 中的依赖，其中已包含打包所需的 `PyInstaller`。

2. 执行打包脚本，默认生成可直接拷贝到其他 Windows 电脑使用的单文件 EXE：

```powershell
python .\build_exe.py
```

或使用 PowerShell 包装脚本：

```powershell
.\build_exe.ps1
```

3. 生成的 EXE 默认位于：

```text
csv_visual\dist\HWiNFO-CSV-Plotter.exe
```

4. 如果希望生成目录模式而不是单文件 EXE，可执行：

```powershell
python .\build_exe.py --onedir
```

5. 如果只想查看将要执行的 PyInstaller 命令，可执行：

```powershell
python .\build_exe.py --dry-run
```

6. `requirements.txt` 同时覆盖 GUI 运行和 EXE 打包所需依赖；默认生成的单文件 `EXE` 适合直接复制到其他 Windows 电脑使用。

## 使用方式

1. 点击“浏览...”选择 HWiNFO CSV 文件
2. 在左侧筛选并多选要绘制的参数
3. 按需设置标题、线宽、时间刻度密度、固定时间间隔、前后裁剪范围、图例、时间/数值刻度文字显示、输出宽高和 DPI，并可为不同参数指定颜色
4. 图表会在选择参数或调整设置后自动预览
5. 点击“导出透明 PNG”

## 说明

- 图表背景为透明，但坐标轴、文字、网格线和曲线会保留
- 预览会在后台线程中按当前宽度、高度、DPI 与样式设置生成内存 PNG，窗口变化不会改写渲染参数
- 预览区会像 Windows 图片查看器一样按窗口大小自动等比 fit，不提供滚动查看原始细节
- 导出 PNG 与预览共用同一套宽高、DPI 和样式配置；预览只改变查看方式，不改变原图内容
- 默认打包产物是单文件 `EXE`，目标电脑无需额外安装 Python；将生成的 `EXE` 文件复制到其他 Windows 电脑即可运行
- X/Y 轴标题输入已移除，时间轴右下角的额外日期偏移标注也已去掉
- 时间轴显示的是抽象时长，而不是 HWiNFO CSV 中的原始日期时间
- 时间轴在小时部分为 `00` 时会自动压缩为 `MM:SS`，超过 1 小时后恢复为 `HH:MM:SS`
- 时间/数值显示开关只控制刻度文字，不会隐藏轴线、刻度线或网格线
- “纯曲线模式”与网格、图例、时间文字、数值文字四个开关互斥；启用后只保留数据曲线，不保留边框、网格、图例和文字
- 可以通过“起始裁剪 / 结束裁剪”滑块去掉前后不需要可视化的时间段
- 可以通过滑块调节时间刻度密度；如果填写了固定时间间隔（秒/分钟/小时），则会优先按固定间隔显示
- 时间轴会在保证可读性的前提下尽量细化，避免短时间范围内网格和刻度过密
- 如果 CSV 中存在重名列，界面会显示 CSV 列号和重复序号来区分
- 如果 CSV 使用 UTF-8、UTF-16、GB18030/GBK 等常见编码，工具会自动选择更合理的解码方式，减少中文乱码
- 如需手工验证，推荐使用你自己的 HWiNFO CSV 导出文件，而不是将真实测试数据写死到自动化测试中
