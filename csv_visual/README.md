# HWiNFO CSV 折线图导出工具

一个面向 Windows GUI 的小工具，用来读取 HWiNFO 导出的 CSV 文件，按时间轴绘制指定参数的折线图，并导出透明背景 PNG，方便接入后续剪辑、排版或可视化流程。

## 功能

- 支持一次添加多个 HWiNFO CSV 文件，也支持直接将多个 CSV 拖入窗口
- 支持移除选中文件、清空全部文件，并自动按规范化绝对路径去重
- 支持四区布局：文件管理、图表预览、参数与图表设置、时间轴
- 支持多个文件在同一时间轴上独立拖动，按需调整每个 CSV 片段的位置来实现想要的对齐画面
- 支持在统一时间轴中拖动 CSV 片段左右边缘，裁剪单个 CSV 的有效片段
- 支持在多文件模式下按“共享参数”进行一次选择，并自动应用到所有已加载文件
- 支持按 `SeriesKey(session_id, column_index)` 维度为不同文件中的参数独立设置颜色
- 支持基于单个共享参数自动检测峰值 / 谷值，并按对齐后的时间轴进行跨文件分组
- 支持在“峰谷映射”区为同一组事件中的不同文件分别填写赋值，并在副 Y 轴渲染派生曲线
- 支持为单个峰谷点单独设置颜色，优先级高于原始曲线颜色
- 支持对齐后的统一时间轴显示；用户可自由拖动文件做时间 offset，界面会自动把最早时间映射为 `00:00`
- 支持后台自动预览与后台数值序列预载，导出 PNG 与预览共用同一套渲染参数
- 支持运行期日志落盘，默认记录程序启动、CSV 读取、对比结构刷新、后台预览 / 预载、导出与异常堆栈
- 支持标题、字体、线宽、图例位置、纯曲线模式、坐标轴/网格/文字颜色、时间刻度密度、固定时间间隔等样式设置
- 支持中文列名、重复列名、列名筛选，以及 UTF-8 / UTF-16 / GB18030 / GBK 等常见编码自动识别

## 架构与扩展

- CSV 日志读取、编码识别、HWiNFO 表头解析、时间戳解析和数值序列缓存已集中在 `hwinfo_plotter/csv_log.py`
- 图表渲染、时间轴对齐、峰谷检测与 PNG 导出保留在 `hwinfo_plotter/core.py`，不再直接承载 HWiNFO CSV 解析细节
- 解析层对外暴露 `CsvLogData`、`SensorColumn`、`CsvLogLoader`、`load_csv_log()` 和 `load_hwinfo_csv()`，后续可为其他应用 CSV Log 注册新的 loader
- 多文件共享参数按 `SensorColumn.shared_key` 匹配；HWiNFO 默认使用“参数名 + 重复序号”，不再要求不同 CSV 中的物理列号一致
- `SensorColumn.source_name` 会保留 HWiNFO 文件尾部的传感器来源描述，便于后续做更精细的设备级匹配或诊断

## 环境

- Windows
- Python 3.10+
- `matplotlib`
- `scipy`

安装依赖：

```powershell
pip install -r requirements.txt
```

## 启动

双击 `run_gui.bat`，或者手动执行：

```powershell
python .\main.pyw
```

- 源码运行时默认写入 `csv_visual\logs\csv_visual.log`
- 打包后的 EXE 默认写入 `EXE 同级目录\logs\csv_visual.log`

- 如需分析问题，优先附上最近一次复现时生成的 `csv_visual.log`

## 使用方式

1. 点击“添加 CSV...”选择一个或多个 HWiNFO CSV 文件，或直接把多个 CSV 拖入窗口
2. 在“文件管理”中查看已加载文件，必要时进行移除、清空或别名设置
3. 文件管理表格保留 `文件名`、`别名`、`时长` 三项信息；下方可直接修改当前选中文件的别名
4. 在“参数与图表设置”中筛选并选择要绘制的共享参数，并按需设置标题、字体、线宽、图例、时间刻度和颜色
5. 在“时间轴”中完成时间编辑：
   - 拖动片段主体：只调整当前抓住的那个 CSV 的 `offset_seconds`
   - 拖动片段左右边缘：调整该 CSV 的原始时间有效片段
   - 时间轴左上角会固定显示 `原点 00:00` 标识；即使某些文件被向左拖动，界面仍会自动保持零起点显示
   - 每个文件的文件名与偏移值会固定显示在时间轴左侧，不会因为水平滚动或缩放而跑位
   - 时间轴默认会按当前窗口宽度自适应显示；可点击“缩小 / 放大”或在时间轴上滚动鼠标滚轮进一步缩放细调
   - 点击“重置时间轴”会把全部已加载 CSV 的偏移、有效片段和时间轴缩放恢复到默认状态，并滚回时间轴起点
   - 时间轴刻度会和图表保持一致，以当前有效片段的对齐结果为准；被裁掉的部分不会继续撑大时间轴范围
   - 默认吸附到 `1s`，按住 `Shift` 拖动可吸附到 `0.1s`
6. 如需生成峰谷赋值曲线，可在“峰谷映射”区：
   - 勾选“启用峰谷映射”
   - 选择 1 个共享参数作为检测源
   - 调整峰 / 谷模式、最小间距、最小突出度、平滑窗口、分组容差
   - 点击“重新检测”查看对齐后的峰谷分组
   - 在表格中选中某一行后，填写赋值或点颜色
7. “图表预览”会自动在后台刷新；原始曲线继续显示在主 Y 轴，赋值曲线默认显示在右侧副 Y 轴
8. 确认结果后点击“导出透明 PNG”
9. 可通过菜单“帮助”→“关于”查看当前版本 Git Hash、仓库、许可证和作者信息

## 多文件对比说明

- 每个 CSV 都维护独立的 `offset_seconds`，不再自动指定单一基准文件；用户可自由输入或拖动正/负 offset，导入、移除或调整某个文件不会重归一化其他文件的偏移量
- 对齐公式为：

```text
aligned_elapsed_seconds = raw_elapsed_seconds + offset_seconds
```

- 单个 CSV 有效片段基于“原始 CSV 时间”裁剪：

```text
source_trim_start_seconds <= raw_elapsed_seconds <= source_trim_end_seconds
```

- 最终参与绘图的数据点会先按该 CSV 的有效片段裁剪，再按 `offset_seconds` 映射到统一时间轴
- 时间轴和图表默认都从 `00:00` 起始；若某个文件被拖到更早的相对时间，界面会自动整体平移显示结果，但不会改变文件之间的相对 offset 关系
- 时间轴范围与图表范围都会优先依据各文件当前“有效片段”的对齐结果计算；被裁掉的前后段不会再额外引入时间偏移
- 多文件模式下，参数列表会按逻辑共享键将所有文件共有的参数去重后仅显示一次；即使同一参数在不同 CSV 中列号不同，用户选择后也会自动映射到各文件对应列
- 参数颜色列表仍按 `[文件别名] [列号] 参数名` 展示，便于为不同文件中的同一参数设置不同颜色
- 文件表格负责列出当前会话文件并维护别名；单文件有效片段统一在时间轴区域中编辑
- 峰谷映射当前只支持一次选择 `1` 个共享参数作为检测源；多文件模式下会自动扩展到所有已加载文件
- 峰谷分组按对齐后的 `aligned_elapsed_seconds` 自动完成：同组内允许不同文件填写不同赋值
- 未赋值的峰谷点不会自动补零；派生曲线会在该点处断开

## 峰谷映射说明

- 峰值检测与谷值检测基于 `scipy.signal.find_peaks`；谷值通过对数列取负后复用同一套逻辑
- `最小间距(秒)` 用于抑制过密极值；`最小突出度` 用于过滤轻微噪声；`平滑窗口` 为 `1` 时表示不平滑
- 多文件分组时不再依赖单一基准文件，组锚点使用同组事件的对齐时间中位数
- 派生曲线按文件分别渲染，例如 `RunA · 赋值曲线`、`RunB · 赋值曲线`
- 单点颜色回退顺序为：峰谷点显式颜色 → 原始参数颜色 → Matplotlib 默认调色板

## 颜色与预览

- 参数颜色支持直接输入 `66CCFF` / `#66CCFF`，也支持使用取色器
- 峰谷映射中的单点颜色同样支持直接输入 `66CCFF` / `#66CCFF`
- 坐标轴、网格、时间文字、数值文字、图例文字都可以独立设置颜色
- 预览在后台线程中按当前宽度、高度、DPI 和样式生成透明 PNG，再在窗口中自适应显示
- 拖动时间轴时会节流刷新预览，释放鼠标后会立即提交一次最终预览
- 导出 PNG 与预览共用同一套样式、颜色、峰谷赋值和 CSV 有效片段配置
- 默认布局采用“左 30% / 右 70%”分区；左列为“上 10% / 下 90%”，右列为“上 85% / 下 15%”，让图表预览保持主视觉区域；时间轴默认自适应当前窗口宽度

## 打包 EXE

### GitHub Actions 自动打包

仓库内置 `Build csv_visual EXE` workflow，会在以下场景自动打包 Windows EXE：

- 推送到 `main` 且变更涉及 `csv_visual/**` 或 workflow 文件
- 推送 `v*` 标签，例如 `v1.0.0`，打包后会自动创建 / 更新对应 GitHub Release 并附加 EXE 与 SHA256 校验文件
- 在 GitHub 网页的 `Actions` → `Build csv_visual EXE` 中手动运行

workflow 会在 `windows-latest` 上安装 `requirements.txt` 依赖，先运行单元测试，再执行：

```powershell
python .\build_exe.py
```

构建完成后，可在该次 workflow run 的 `Artifacts` 中下载 `HWiNFO-CSV-Plotter-windows-<run_number>`。手动运行时可选择 `onefile` 或 `onedir` 输出模式；默认是单文件 EXE。打包脚本会在构建时写入当前 Git Hash，并把 `about.md` 打入 EXE；输出文件 / 目录名也会自动追加对应的 `7` 位短 Git Hash，例如 `HWiNFO-CSV-Plotter-25433f8.exe`，应用内“关于”窗口的 `Version` 会随打包提交自动变化。

### 本地打包

1. 确保已安装 `requirements.txt` 中的依赖，其中已包含打包所需的 `PyInstaller`
2. 执行：

```powershell
python .\build_exe.py
```

或使用 PowerShell 包装脚本：

```powershell
.\build_exe.ps1
```

3. 默认产物位于：

```text
csv_visual\dist\HWiNFO-CSV-Plotter-<short_hash>.exe
```

4. 如需生成目录模式而不是单文件 EXE，可执行：

```powershell
python .\build_exe.py --onedir
```

5. 如只想查看将要执行的 PyInstaller 命令，可执行：

```powershell
python .\build_exe.py --dry-run
```

## 说明

- 图表背景为透明，但坐标轴、文字、网格线和曲线会保留
- 预览区会按窗口大小自动等比 fit，不提供滚动查看原图细节
- 纯曲线模式会关闭网格、图例、时间文字和数值文字，只保留数据曲线
- 时间刻度会在保证可读性的前提下尽量细化；若填写固定时间间隔，则优先使用固定间隔
- 时间轴中的“重置时间轴”会把全部已加载 CSV 的偏移、有效片段和时间轴缩放恢复默认，并把水平滚动位置回到起点
- 时间轴不再提供全局可视范围裁剪，预览 / 导出会按当前文件有效片段与对齐结果完整显示
- 源码运行时“关于”窗口会优先读取当前仓库 Git Hash；打包 EXE 会使用构建时注入的 Git Hash
- 若需要手工验证，建议使用你自己的 HWiNFO CSV 导出文件，避免把真实数据固化到自动化测试中
